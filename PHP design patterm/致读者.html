致读者：

我是今年 2 月份第一次读到这本书的，当时我是想找一本系统介绍在 PHP 中实现设计模式的书。于是我找到了 php|architect’s Guide to PHP Design Patterns。说实话，现在 PHP 这方面的经典书籍还真不多。虽然我的英语水平还算马马虎虎，不过看完这本书还是花了我不少力气。

对该书的评价这里暂时不说了。我要重点强调我翻译本书的目的。

正如我前面说到的，看英文书确实是一件费劲的事情，如果你和我一样，英文不是特别牛的话。(不过我还是强烈推荐多看英文原版书籍。)所以后来我决定把这本书翻译出来。不过很遗憾的是我没有上 phpChina，不知道什么时候他们已经开始在组织翻译了。等到我初稿完成的时候，我偶然在网上发现已经有了该书的中文版本。其实初稿并没有花费我太多的时间，只是中间有一段时间我几乎没有在网上逗留过。于是，我把它们撂在了一边。

最近，我粗略地看了一下 phpChina 的版本。虽然我们应该高度赞扬他们的无私奉献精神，不过我还是发现有很多不足之处。也许是因为由不同的译者翻译的缘故，也可能是因为缺乏一个统一的标准。此外，还有因为一些别的原因，所以我重新整理了一下初稿，现在把它们公布出来。

我看过很多英文书籍的中文电子版本，且不说它们的翻译质量，我几乎没有见到过排版良好的电子版本，几乎清一色的都是翻译出来了事。而很多英文版电子书在这方面就做得很出色。当然，我们应该谅解这种情况，因为毕竟翻译是一件劳心劳神劳力的工作，本来就是一种奉献的精神，还要要求我们的译者把最后的成果排版得很好似乎苛刻了一点。

不过我是一个有些完美主义倾向的人，我总是希望能够看到更好的成果。而且我相信，之所以会出现这种混乱的情况，之所以组织好中文电子版的结构如此困难，主要还是因为我们缺乏一个比较好的标准。因此，我想借翻译本书的机会，能够与大家一起讨论这方面的问题。其实这件事情也不是很难，因为我们只要稍微看看一些英文电子书籍的组织方式，看看它们的 css 文件，我们很快就能形成我们的标准了。不过说实话，我对发起这次讨论并没有多大的信心，因为我感觉这是一件有些乏味的事情，我不敢肯定会有人肯来与我一起讨论，不过我还是衷心的希望能够得到大家的帮助。

在整理本书的过程中，我在少量的几个地方参照了 phpChina 版的翻译，因此，我在这里要特别对他们表示感谢。其实，我之所以要发布我翻译的这个版本还有另一个原因就是提高自己阅读英文书籍的能力。我发现，试着翻译英文文章是提高这项能力的非常不错的方法。所以如果你也希望这么做的话，那么我建议你把英文版拿过来，慢慢地，对照你自己的翻译，我的翻译，phpChina 的翻译，相信最后你能得到很大的收获。特别是对于那些正在学习，需要过级的的朋友，我想也许这也是一个不错的方法。

因为我的英文水平，中文水平，编程知识等很多方面的能力有限，本书翻译的很多地方还是存在很大的缺陷。所以我希望，如果你有什么建议或意见都可以告诉我，哪怕是你认为在什么地方应该多增加一个空格或空行，什么地方应该增加一个逗号等，都可以告诉我，我将非常感激你们的赐教。毕竟，更好的排版也是我翻译本书的目的之一。当然，如果你能提出更好的，更通顺，更自然，更有艺术性的翻译就更好了。如果你不想麻烦，你也可以集中到一个文件里面，然后一起发给我。

现在的版本还不是最终的版本，我自己对很多地方都还不是特别满意，因此我只是用的 html 文件的方式，而没有把它们组织成 chm 文件。在以后一段时间内，我还会继续改进，如果能够得到大家帮助的话，我想改进的速度会更快。如果你想在你的收藏夹中保留本书，那么你应该过一两个月再次下载本书，我想，到那时应该是一个比较完善的版本了。

我在翻译本书的过程中还做了很多其它工作，包括用 xhtml 来组织文档，重新组织代码结构等。总之，你有什么都可以跟我说。另外，我也希望结识很多对翻译英文文章，或想提高英文水平，或对编程感兴趣的朋友，特别是那些有丰富经验的朋友，希望得到你们的指教。
